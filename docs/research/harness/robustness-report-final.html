Code-Understanding Tool Robustness — cross-repo final report

7 static code-navigation tools, the same 20-anchor "where is this symbol used?" probe, replayed across 3 repositories / 2 languages. Each round scored against an independently-computed, binding-resolved gold (Codex-reviewed every round; R2/R3 golds also Opus-audited; every round's findings adversarially Codex-reviewed). The question: which tool verdicts are durable capability laws, and which were artifacts of one repo, one build, or one language?

5 · What GENERALIZED — durable capability laws

① Node-type + reference-style governs graph-tool blindness

The single biggest discriminator. CBM is the clean experiment: TS schemas as const/Variable used by member-access (paperclip Drizzle) → 0.08; the same tool on Python Pydantic/SQLAlchemy (class → Class nodes w/ USAGE/IMPORTS/INHERITS edges) → 0.95 / 0.97. The blindness tracked the language idiom, not the tool — hence CBM's 0.63 → 0.95 → 0.85 swing.

② Safety hazards are tool-intrinsic — all 3 repos

graphify drops "secret"-named files: on OpenHands (stored_custom_secrets.py→A13 Org) and DeerFlow (test_mcp_config_secrets.py→A7 ExtensionsConfig) a dropped file referenced a live anchor, measurably corrupting recall. CBM git-bleeds to the enclosing worktree's HEAD on all 3. Both are deployment risks independent of language.

③ Lexical = guaranteed recall, collision-precision collapse

ast-grep & repomix recall 0.88–1.00 every round, but collision precision craters (paperclip P .28 / .26). ast-grep's one structural edge is the language split (-l python/-l tsx) — it alone avoids cross-language name bleed (R2 A12 Settings: P=1.0 where graph tools mix Python & TS).

④ Polymorphic / method dispatch is hard for graph tools

Reproduced on both Python repos: graphify's call-graph is same-file-only (DeerFlow A20 F1 0.12), codegraph merges same-named methods, serena's pyright has method-resolution gaps. No graph tool resolves runtime dispatch; lexical over-recalls it. The method bucket is depressed for everyone, every round.

7 · The collision frontier — the open problem

Collisions are the hardest bucket every round — but how hard is repo-dependent, and no tool is reliable across collision types.
  • The floor moves with the repo: OpenHands' milder collisions let CBM reach 0.95 and repomix 0.73; DeerFlow's harder collisions hold every tool ≤ 0.55; paperclip sits between (serena 0.78).
  • What transfers is the ranking: binding/LSP tools (serena, CBM) beat name-keyed graph/lexical tools, which merge same-named symbols.
  • The standout: only serena's LSP cleanly disambiguates the hard cross-language collision — DeerFlow A17 Message (local React component vs @langchain/langgraph-sdk import) → serena 1.00, every other tool ≤ 0.14 (ag .14 / cg .10 / CBM .12 / rpx .05 / UA 0).
  • …yet serena itself loses DeerFlow's Python-method collisions — A5 LocalSandbox.execute_command and A12 StreamEvent → 0.00.
Disambiguation is the open frontier: no tool wins every collision type, on any repo.

9 · Method, isolation & honesty

Gold. 20 anchors/round across 9 buckets; a verdict requires reproduction on ≥2 anchors per bucket. Each gold is binding-resolved (import/re-export-closure, two-root + path-alias aware) with a zero-unaccounted invariant; method anchors (polymorphic) have inherently capped precision. R2/R3 golds were dual-reviewed (independent Opus auditor + Codex gpt-5.5 xhigh, converged); R1 was Codex-reviewed.

Battery. Each tool built a fresh index of each repo, ran blind (no answer key), scored independently. On R3 a session-limit interruption killed the first worker batch mid-run; 6 tools were re-run deterministically via shell (reusing each worker's adapted runner) and UA via a worker — isolation audit confirms every scored number came from a cleanly-isolated run (tool-local HOME, telemetry off, 0 fresh real-HOME writes).

Honesty. F1-overall flatters the lexical tools — every claim here is grounded in the per-bucket + precision/recall split. Collision absolute scores are repo-dependent and not cross-round comparable; the ranking transfers, the numbers do not. Every round's findings were adversarially Codex-reviewed and corrections applied before these figures were used. Raw data, golds, scorers live gitignored in scratchpad/harness-{paperclip,openhands,deerflow}/; per-round detail in the three findings docs + the synthesis.
Bottom line: there is no single best tool — there are capability laws. Graph-tool coverage tracks (extractor × language idiom); binding-resolution buys precision at the cost of idiom-sensitive recall; lexical buys recall at the cost of collision precision; safety hazards (secret-drop, git-bleed) are tool-intrinsic; and collision disambiguation is unsolved. A single-repo benchmark would have mistaken at least two build/language artifacts for permanent tool verdicts.
