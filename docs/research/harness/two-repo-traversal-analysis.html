Code-Traversal Tool Analysis

bodha + data-miner · measured, Codex-gated study of which code-understanding tool best supports traversing and understanding the user's two Python codebases.

36 anchors (18 + 18)4 tools scored 21 symbol-type bucketsAST gold, Codex-audited 3 Codex passes
Bottom line. For plain file-level "which files touch this name," all four tools are near-identical (0.98–1.0) — pick the fastest, ast-grep. The real differentiator is disambiguation — "which definition does a usage bind to" — which matters intensely in data-miner's auto_annotation_v2/v3/v4 dirs that redefine the same class names 3–4×. There serena is the clear winner (0.99), with CBM a strong but imperfect second (0.84).

4 · Per-tool verdicts

serena — the precision / disambiguation leader

Clean disambiguation (0.995); the only tool that resolves import aliases (KnowledgeAuditResult as AgentAuditResult → correct def, verified). Two structural limits: (1) needs an in-index definition — for a base class defined in a library (AbstractCapability from pydantic-ai, with no def-anchor supplied) it returns nothing; (2) find_implementations unsupported in this pyright build.

CBM — strong all-rounder; the pilot verdict softened

Best merged among binding tools (0.997). Softening the 4-anchor pilot (which found CBM collapses same-named classes): on these repo-wide, git-rooted indexes it does not simply collapse — its file_path-scoped queries give different per-def sets (0.84). But the separation is imperfect: cross-version over-attribution (BoundingBox v3/v4 overlap on manual_reviewer/ files) and it misses the alias case (BO2 0.50). Best on the method-dispatch anchor (recall 0.83 vs serena 0.50). git-bleed neutralized (both repos own .git).

codegraph — consistent, with two recall caps

Solid merged (0.98); per-def scoping works but its trail truncates at ~11 entries, capping recall on high-fan-in collisions; missed one local-import use on ContextPack; misses the alias (0.50). The safe zero-setup default, not the precision tool.

ast-grep — the honest lexical ceiling

Perfect merged recall (1.0) in 0.5 s, and the only tool that finds usages of an externally-defined symbol (AbstractCapability: ast-grep 1.0, serena/CBM 0.0). Cannot disambiguate (0.49) — orphan-sibling hits become false positives.

5 · Capability laws — confirmed & revised

Confirmed on the user's own code

  • Binding/LSP tools disambiguate collisions; lexical cannot (serena 0.995 vs ast-grep 0.49).
  • serena uniquely resolves import aliases to the right definition.
  • Method/polymorphic dispatch is the hardest bucket for graph tools (codegraph recall 0.33).
  • Merged where-used (file-level, def-excluded) is essentially solved by any tool.

New / revised on these repos

  • CBM disambiguates nontrivially (0.84) — softens, not overturns, the pilot's "collapse": it separates defs but over-attributes across versions and misses aliases.
  • Externally-defined symbols invert the ranking — for this setup (def outside snapshot, no external anchor supplied): only lexical/name-keyed tools find usages. Dependency indexing could change this.
  • codegraph's per-def truncation is a concrete recall ceiling.

7 · Honesty & caveats

  • The merged near-tie is metric-specific — file-level use-sites, def-files excluded. Including def-sites or scoring at line level would lower the binding tools vs lexical.
  • n is small per bucket (1–4); collisions/clean-models have ≥2 anchors. Single run; per-tool outputs not dual-reviewed. File-level scoring only.
  • Two serena 0.0 anchors are artifacts: AbstractCapability (external def, no anchor supplied — setup-specific, §5) and YOUTUBE_DOMAINS (a module constant the gold-builder didn't register as a def → no def-site to query). Excluding both, serena merged = 0.984.
  • Gold gaps (Codex-flagged): star re-exports (from .settings import *) not traced → DM16 VLMConfig dropped from the disambiguation mean; attr_caller = attribute refs, not strictly call-sites.
  • Two scoring bugs found & fixed via Codex review: (a) serena/codegraph per-def dict shapes were dropped to a merged fallback (had inflated CBM's relative lead); (b) the per-def module→def mapping didn't normalize src/ layout, which had silently dropped all three bodha collisions, and orphan siblings weren't FP-scored. All §2/§3 numbers are post-fix. The serena > CBM ≈ codegraph > ast-grep ranking held across the fix — the validation Codex required.
