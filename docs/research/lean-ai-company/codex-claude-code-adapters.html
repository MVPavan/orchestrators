Codex and Claude Code adapter architecture

Consolidated sub-agent research on building a first coding department around Codex and Claude Code, after the FastAPI, React/Vite, Pydantic AI, LiteLLM, Temporal, and app-owned Postgres decisions.

CLI first Use Codex and Claude Code as external runtime adapters, not embedded product state.
One writer One isolated worktree per implementation attempt; critics stay read-only.
Git is truth Capture authoritative diffs, changed files, untracked files, and test logs outside model prose.
App owns safety Approvals, secrets, artifacts, audit, cancellation, and merge decisions stay app-owned.
01

Answer

Recommendation: build a CodingRunWorkflow in Temporal with app-owned Postgres records, per-run git worktrees, and two initial runtime adapters: CodexLocalExecAdapter and ClaudeCodeAdapter.

Default substrate: subscription-compatible subprocess adapters over stable CLI surfaces. Use Codex exec/exec resume/exec review for Codex. Use Claude Code CLI claude -p as the Claude default when subscription login is a hard requirement; use Claude Agent SDK only when API-key billing is acceptable or Anthropic-approved subscription-product auth is available.

The adapter boundary should not be "send prompt, parse final text." It must cover task handoff, workspace lease, role policy, runtime invocation, streaming event capture, approval bridging, independent verification, patch provenance, artifact retention, and cancellation cleanup.

Do not make the Claude Code Codex plugin, raw Codex app-server, interactive slash commands, API-only SDK paths, or a shared desktop CLI login the core product substrate. Keep those as convenience or later integration paths. The v1 system should be boring: Temporal orchestrates, Postgres records, git worktrees isolate, subscription-compatible CLI runtimes execute, and Git/test artifacts decide.

02

Context And Method

Research was split across four sub-agents: current Codex surfaces, current Claude Code surfaces, workspace/safety/artifact architecture, and red-team synthesis. Source weighting favored official Codex, Claude Code, Git, and Docker docs plus verified local CLI help. Prior repo decisions were treated as constraints.

Prior DecisionAdapter Consequence
Temporal is the durable workflow engine. CLI invocations are activities/attempts under CodingRunWorkflow; local CLI session state is not durable product state.
Postgres owns product/audit state. Runs, approvals, workspaces, invocations, diffs, patches, tests, artifacts, and secret grants are app records.
React + Vite is the cockpit. UI consumes normalized run events, patch review, approval requests, artifact previews, and verification results.
Pydantic AI is default in-process SDK. Pydantic AI can coordinate or classify, but Codex and Claude Code remain external execution lanes for code work.
Subscription-first when practical. Prefer local Codex CLI and Claude Code CLI subscription-OAuth surfaces before API-only SDK implementations, while preserving clean adapter exits.

Workflow Contract

StageOwnerRequired ArtifactsSafety Rule
Task packet FastAPI + Postgres Objective, repo/ref, allowed paths, non-goals, role, verification command, budget. Immutable after run start; revisions create new attempts.
Workspace lease WorkspaceManager Base SHA, worktree path/container ID, dirty-state snapshot, writer lock. No two writer runtimes share one workspace.
Spec review Codex or Claude critic Read-only stream, final critique, cited-file verification notes. No edits; no shared context between independent critics.
Implementation Exactly one writer runtime Runtime stream, stdout/stderr, command events, changed-file manifest. Workspace-write only inside leased workspace; approval for network/secrets/destructive actions.
Verification Supervisor activity Declared test command, exit code, logs, parsed counts where available. Trust supervisor-run tests, not model prose.
Patch capture Git collector git diff --binary --full-index, diff stat, untracked artifacts, diff hash. Git is authoritative patch provenance.
Critic review Other runtime, read-only Patch review, risks, missing tests, accept/reject recommendation. Critic cannot edit the implementer workspace.
Apply or discard Human/app policy Approval decision, applied patch hash, merge/branch result, cleanup status. No commit, push, dependency install, or secret grant without explicit policy/approval.
06

Data Model

Core Records

  • coding_run: adapter, role, repo, base ref/SHA, branch, workspace, isolation mode, status, parent run.
  • task_packet: objective, non-goals, allowed paths, runtime role, verification command, max budget.
  • workspace_lease: before/after heads, dirty snapshot, changed files, submodule status, cleanup policy.
  • runtime_invocation: argv, cwd, CLI/SDK version, model/profile, permission/sandbox mode, env allowlist hash.
  • invocation_event: normalized JSONL/stream-json/stdout/stderr chunks, tool events, command events, redaction state.

Evidence Records

  • patch_provenance: base SHA, diff hash, changed files, untracked capture, prompt hash, attempt ID.
  • test_run: command, cwd, env profile, exit code, duration, stdout/stderr artifact, parsed counts.
  • approval_request: requested capability, reason, risk class, scope, reviewer, decision, expiry.
  • secret_grant: alias, purpose, injection mode, start/end, redaction rule; never store values.
  • artifact: kind, path/blob key, SHA-256, size, MIME, redaction status, retention policy.

Failure Modes

RiskSeverityMitigation
Plugin internals or interactive slash commands become the product contract. High Use documented subscription-compatible CLI surfaces for adapters. Keep Claude Code Codex plugin and TUI flows as human convenience paths.
API-key-only SDK path silently becomes the default and violates the subscription-first constraint. High Gate every adapter with an auth-mode field. Default Codex to ChatGPT-managed CLI auth and Claude to subscription OAuth CLI auth; require explicit approval for API-key fallback. For Claude, reject --bare when auth_mode=subscription_oauth.
Two writer agents mutate the same checkout. High Use one writer lock per workspace. Critics read captured patches or read-only worktrees.
Stdout-only parsing hides partial failures, commands, approvals, or tests. High Capture native structured streams plus raw stdout/stderr, process exit code, git diff, command logs, and supervisor tests.
Secrets leak through prompts, env dumps, shell traces, diffs, or logs. High Use no full host env, scoped secret aliases only, redaction before persistence, path denies, and explicit approval for secret grants.
CLI permissions are treated as a full sandbox. High Combine worktrees with permission modes, hooks, minimal env, and container isolation for privileged/untrusted/networked runs.
Cancellation leaves a dirty workspace that is retried or merged later. Medium After cancellation, capture partial artifacts, mark workspace quarantined, and retry in a fresh workspace.
Fake debate shares context and produces anchoring instead of independent critique. Medium Run independent reviewers from the same immutable task packet; reveal prior opinions only after verdict capture.
Central server multiplexes one desktop CLI login across users or tenants. High Use per-user local runners or dedicated scoped service credentials. Do not store raw CLI auth tokens in product DB.

Upgrade Triggers

NeedUpgrade PathWhy Not First
Mid-turn steering, richer persistent thread state, or lower-level event control for Codex. Codex SDK first; raw app-server only if SDK/MCP cannot expose the required feature. CLI JSONL is enough for v1 and app-server is an experimental/deep-integration surface.
Programmatic Claude approvals and user input with durable resume. Claude Agent SDK only when API-key billing is acceptable, Anthropic-approved subscription auth exists, or future official docs explicitly support this product path. Current SDK docs are API-key oriented for product builders; subscription-first v1 should wrap claude -p instead.
Untrusted code, broad network access, dependency installs, or bypass permissions. Container/VM runner with restricted mounts, non-root user, network controls, and no Docker socket. Git worktrees are isolation for diffs, not a security boundary.
Remote branch/PR work or parallel offload. Codex Cloud or Claude GitHub Action as sidecar adapters. They split artifact ownership and require environment/GitHub policy setup.
Multiple repos or team/tenant execution. Runner fleet with per-user or per-service credentials, workspace quotas, and artifact retention policy. Do not start by sharing one local CLI auth across all runs.
