Data And State

State is split across environment-derived configuration, SQL session stores, file or object-backed event bodies, file or SaaS-backed settings and secrets, pending-message queues, live sandbox state, and enterprise tenant metadata.

Configuration State

Environment firstConfiguration is loaded through SDK env mapping with OH prefix, then legacy variables such as OH_PERSISTENCE_DIR, FILE_STORE_PATH, WEB_HOST, LLM_BASE_URL, and search keys are handled.
Default persistenceWhen no persistence path is supplied, config defaults to ~/.openhands and creates the directory during defaulting.
Runtime-dependent behaviorSandbox service and event storage are selected from config, so source defaults are not enough to predict a deployment without env values.

Conversation And Task State

Conversation metadata

OSS V1 metadata lives in conversation_metadata through StoredConversationMetadata. It stores conversation id, sandbox id, repository, branch, provider, title, trigger, PR numbers, LLM model, agent kind, metrics, parent id, public flag, tags, timestamps, and conversation_version.

Refs: external/OpenHands/openhands/app_server/app_conversation/sql_app_conversation_info_service.py:65, external/OpenHands/openhands/app_server/app_conversation/sql_app_conversation_info_service.py:347.

Start task state

Start tasks store task status, detail, eventual conversation id, sandbox id, agent-server URL, original request, and timestamps. They make frontend polling possible while backend startup continues.

Events, Settings, Secrets, And Pending Messages

State familyOwnerImportant behavior
Event bodiesEventService abstraction over filesystem, S3, or GCS.Filesystem writes use temp file plus replace; callback execution is SQL-backed and captures processor exceptions as stored error results.
SettingsOSS file store or SaaS settings store.Generic updates reject raw agent/conversation-settings replacement and ignore generic llm_profiles updates.
SecretsOSS JSON file or SaaS encrypted custom secrets per user/org.OSS drops provider-token entries without token; SaaS deletes/reinserts effective-org custom secrets on store.
Pending messagesSQL-backed queue keyed by conversation id.Queued messages sent to task-{uuid} are rewritten to the real conversation id after startup and deleted regardless of per-message success.
