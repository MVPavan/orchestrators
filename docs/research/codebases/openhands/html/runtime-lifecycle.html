Runtime Lifecycle

The local GUI starts quickly by returning a start task, then keeps the backend conversation startup generator alive in the background while sandbox creation, workspace setup, SDK request construction, agent-server conversation creation, and pending message replay complete.

Lifecycle Facts

ASGI compatibility entryRun commands still point at openhands.server.listen:app, but that module re-exports openhands.app_server.app.
Frontend returns a taskuseCreateConversation() receives a start task and the route polling hook navigates only after the task becomes READY.
Startup is backgroundedPOST /app-conversations returns the first task and consumes the remaining async generator after the HTTP response.

Repository And Workspace Setup

Project root decision

Selected repositories resolve to {working_dir}/{repo_name}. Empty workspaces use the configured working directory directly.

Setup stages

  • PREPARING_REPOSITORY: create workspace, initialize git, or clone with authenticated credentials.
  • RUNNING_SETUP_SCRIPT: run .openhands/setup.sh if available.
  • SETTING_UP_GIT_HOOKS: install OpenHands pre-commit hook while preserving an existing local hook.
  • SETTING_UP_SKILLS: ask the agent-server to load public, user, project, org, and sandbox skills.
