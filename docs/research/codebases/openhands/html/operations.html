Operational Model

Operationally, this repo has heavy dependency, Docker, and environment-sensitive paths. For this architecture research task, OpenHands runtime commands were not executed; verification stays focused on generated docs and source-reference sanity in the parent repository.

Docker And Packaged App

Packaged app

docker-compose.yml builds containers/app/Dockerfile, exposes port 3000, mounts the Docker socket, maps ~/.openhands to /.openhands, and mounts the workspace base into /opt/workspace_base.

Refs: external/OpenHands/docker-compose.yml:2, external/OpenHands/docker-compose.yml:24.

Runtime image

The app Dockerfile uses frontend-builder, backend-builder, and runtime stages. It sets defaults including RUN_AS_OPENHANDS, SANDBOX_LOCAL_RUNTIME_URL, WORKSPACE_BASE, FILE_STORE=local, and INIT_GIT_IN_EMPTY_WORKSPACE=1.

Enterprise Operations

Enterprise has separate setup and run instructions. The repo guidance names Python 3.12, Poetry, Node.js 22.x, and optional Docker, then instructs developers to build the main project, install enterprise dependencies, and install enterprise pre-commit hooks. Enterprise tests use poetry run --project=enterprise pytest with xdist/forking and enterprise-specific config; server startup uses enterprise make targets.

Refs: external/OpenHands/AGENTS.md:174, external/OpenHands/AGENTS.md:196, external/OpenHands/enterprise/saas_server.py:81, external/OpenHands/enterprise/server/app_lifespan/saas_app_lifespan_service.py:19.

Operational Risk Signals

SignalReason it matters
Dependency and runtime commands are heavy.Build paths install backend and frontend dependencies, install hooks, and may build/pull containers.
Runtime behavior is environment-sensitive.Sandbox mode, DB backend, event storage, CORS, web URL, and provider base URL depend on runtime configuration.
Packaged local GUI can access Docker socket.This is expected for sandbox container management but increases operational blast radius.
Enterprise overrides can be brittle.The enterprise README explicitly warns about middleware stacking and user-id versus token assumption drift.
