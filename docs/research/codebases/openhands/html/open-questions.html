Open Questions

This page separates unresolved source questions, explicit inferences, intentionally unaudited surfaces, and recommended next reads so future agents do not overstate the architecture pass.

High-Signal Open Questions

QuestionStatusWhy it mattersSuggested follow-up
What exactly lives inside the pinned SDK, agent-server, and tools packages?OpenThis repo builds requests and calls the agent-server, but agent execution semantics live behind that package boundary.Inspect matching package source or vendor contents for version 1.28.0.
How do deployed environment values differ from defaults?OpenRuntime choice, DB backend, storage provider, CORS, web URL, provider base URL, and sandbox mode are env-sensitive.Collect representative OSS local and SaaS/Enterprise configs with secrets redacted.
Is OSS conversation metadata intentionally unscoped by user?Partially answeredEnterprise adds explicit user/org filtering; OSS service filtering appears V1-only in sampled paths.Confirm the OSS threat model and intended deployment assumptions with maintainers or tests.
What is the full remote runtime API contract?OpenRemoteSandboxService adapts a legacy runtime HTTP protocol, but this pass traced only the app-server adapter.Read remote runtime docs/source and exercise start/list/pause/resume/delete with a test server.
Which frontend workflows still rely on direct runtime URLs?Partially answeredDirect runtime calls require correct conversation_url, session API key handling, and browser reachability.Search all buildRuntimeUrl, direct axios, and conversation_url uses.
Are enterprise provider callbacks tenant-scoped consistently?OpenEnterprise adds user/org scoping for conversations and pending messages, but provider callbacks can add separate paths.Review provider-specific callback processors and enterprise unit tests.
How stable is V1 versus V0 terminology?OpenThe skills README says V1 skills are not yet released while code is V1-heavy and wrappers remain.Check current release notes/docs and normalize downstream terminology accordingly.

Claims That Are Inferences

Orchestration shellThe local GUI appears to be an orchestration shell around SDK/agent-server packages, inferred from package pins, README boundaries, and code that POSTs SDK request JSON to the sandbox agent-server.
OSS tenancyOSS appears single-user or weakly multi-tenant compared with SaaS. This is an inference, not a security verdict.
Thin runtime proxyThe app-server appears intentionally thin around running-agent actions, inferred from send-message routing notes and frontend direct runtime URL helpers.

Areas Deliberately Not Audited

  • Full component-by-component frontend behavior and visual design.
  • Full provider implementation internals across GitHub, GitLab, Bitbucket, Jira, Linear, Slack, Azure DevOps, and Enterprise SSO.
  • Every Alembic migration; only V1 and tenant-related migrations were sampled.
  • Real Docker/runtime execution, dependency installation, browser workflows, and network calls.
  • Security review of token encryption, JWT details, provider token refresh, callback auth, or API key storage.
  • Performance/capacity analysis of polling, DB indexes, sandbox limits, and event storage.
