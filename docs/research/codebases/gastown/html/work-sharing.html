Work sharing

Work is shared by assigning a durable work item to one worker and then starting that worker session.

The assignment is not a chat message. It is a Beads state transition plus worker metadata. The terminal prompt only tells the worker to inspect its current assignment.

Assignment To Completion Flow

What Dispatch Writes

WritePurpose
Work item status and assigneegt hook materializes the assignment with bd update <id> --status=hooked --assignee=<agent>.
Agent hook_beadThe worker's agent record points back to its current work item for status displays and recovery logic.
Work item fieldsDispatch metadata such as dispatcher, args, formula, attached molecule, no-merge, review-only, mode, and formula variables.
Repo branch/worktreeThe worker gets an isolated workspace branch so implementation changes are separate from the integration branch.
Activity eventA feed entry records that work was assigned to the worker identity.

How Contention Is Controlled

One lock per work item

Dispatch takes a flock for the target work item so simultaneous dispatch paths do not both assign it.

One lock per assignee

Before writing the assignment, dispatch serializes writes for the chosen worker so concurrent hook updates do not overwrite each other.

Hook replacement rules

If the worker already has a hooked item, replacement is allowed only when complete, explicitly forced, or auto-forced because the old session is dead.

Stolen work shutdown

When forced reassignment steals work from a live worker, dispatch sends lifecycle shutdown mail to the old worker's witness.

Completion Path

StageInternal behavior

Workspace safetyIt protects stash state, auto-commits safe uncommitted work, blocks unsafe default-branch completion, fetches, rebases, and checks contamination.
SubmissionCompleted work pushes the branch and creates or reuses a merge-request work item unless the item is no-merge, review-only, escalated, or deferred.
TraceabilityThe worker agent record gets completion metadata such as exit type, MR id, branch, last source issue, and completion time.
Merge processingThe merge processor reads open MR work items, runs checks, merges or reports failure, closes the source work on success, and nudges relevant roles.

Important Takeaway

Work is shared through Beads records and git branches. Terminal messages only wake agents up; the source of truth is the assignment state and repo state that any replacement or recovery session can inspect.
