Agent communication

Agents communicate through durable mail and queued or direct terminal nudges.

The system does not create direct agent-to-agent sockets. It writes messages into durable state, resolves a recipient identity to one or more sessions, and optionally wakes those sessions through tmux.

Communication Paths

Three Communication Lanes

Mail

Router.Send expands lists or groups, validates the recipient, writes a Beads-backed message, then starts notification in the background.

Nudge

gt nudge or mail notification sends a short reminder into the recipient session. If the agent is busy, the nudge is queued for later delivery.

Lifecycle

Lifecycle subjects such as LIFECYCLE:Shutdown are mail messages that tell a role manager or witness to stop, restart, or recover a session.

Mail Internals

OperationInternal behavior
Address routingRecipient strings are resolved against town roles, repo roles, crew, workers, route files, and fallback workspace directories.
Durable writeThe router builds a bd create command with assignee, body, priority, labels, actor, and subject. The message exists even if notification fails.
Fan-outLists and groups create individual message copies for each recipient.
NotificationAfter the durable write, notification is asynchronous. The router waits briefly for idle; idle sessions receive a direct nudge, busy sessions get queued.

Nudge Internals

CaseDelivery detail
Idle tmux sessionGastown serializes access to the pane, exits copy mode, sends literal text chunks, presses Enter, and wakes the pane.
Busy tmux sessionThe nudge is written as JSON in .runtime/nudge_queue/<session>/ and drained later.
Runtime without hook supportA background nudge poller checks the queue, waits for idle, drains due files, and injects them through tmux.
No tmux pane or ACP modeDelivery falls back to the queue because there is no interactive terminal pane to type into.

Queue Semantics

Nudge files use timestamp-based names for FIFO ordering. Draining atomically renames each file to a unique claimed path so two drainers do not deliver the same nudge. Expired nudges are deleted; deferred nudges are unclaimed and left for a later drain.
