Core architecture

Gastown is a state coordinator around agent CLIs, git, Beads, and tmux.

There is no central in-process orchestrator that owns all agent state. The gt command changes durable state, starts or nudges terminal sessions, and lets agents recover context from the same stores.

Component Responsibilities

gt command layer

Runs the transitions: dispatch work, hook a work item, start sessions, send mail, submit completion, and wake the merge processor.

Agent managers

Prepare directories and settings, build the environment, construct the agent startup command, and create the tmux session.

Worker sessions

Read current assignment, modify the repo worktree, and call gt done when complete. They do not own the global queue.

Merge processor

Reads merge-request records, validates branch state, runs configured gates, merges, pushes, and closes source work on success.

Core Mental Model

Gastown shares work by mutating durable records, then making sure the right terminal session notices. That is the main internal rule behind dispatch, mail, nudges, completion, and merge processing.
