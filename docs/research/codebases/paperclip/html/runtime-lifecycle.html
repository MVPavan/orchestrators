Lifecycle Phases

Startup

startServer() waits for instrumentation, loads config, configures secrets, applies migrations, creates DB clients, builds auth/storage/plugin services, starts WebSockets, and starts recovery loops. Source: external/paperclip/server/src/index.ts:102-116, 151-199, 642-704, 753-895.

HTTP/API

createApp() installs middleware, auth, route groups, plugin routes, adapter routes, static plugin UI, built UI serving, and Vite dev middleware. Source: external/paperclip/server/src/app.ts:129-437.

Wakeup

The heartbeat service checks company/agent state, budget, policy, issue status, dependency readiness, holds, coalescing, and queued run insertion. Source: external/paperclip/server/src/services/heartbeat.ts:10099-10795.

Run Execution

A queued run is claimed per agent, then context, workspace, environment lease, runtime services, adapter execution, logs, usage, and result state are processed. Source: external/paperclip/server/src/services/heartbeat.ts:7666-9025, 9120-9205.
