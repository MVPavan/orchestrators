# Research Notes

Durable research notes live here when they inform project direction but are not yet formal plans,
workstreams, or implementation docs.

Use this folder for:

- runtime and provider comparisons
- current-docs research summaries
- orchestration design investigations
- tradeoff notes that may later become workstream inputs

Keep notes dated, source-grounded, and repo-relative. If a research note turns into execution work,
promote it into `docs/brainstorms/` or `docs/workstreams/` instead of treating research as the plan.
